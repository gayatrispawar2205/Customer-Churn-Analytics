Customer Churn Analytics & Prediction

End-to-end customer churn analysis using SQL for business insights and Python for machine learning, built as part of a data analytics portfolio.

Project Overview

This project focuses on understanding why customers leave a service (churn) and identifying customers who are at high risk of churning. The goal is to combine SQL-based analysis with Python-based machine learning to deliver actionable business insights.

Business Questions Answered

Why do customers churn?
Which customer segments have the highest churn?
How do contract type and payment method affect churn?
Which customers are at highest risk of churning?

Workflow

1. Data Validation

Cleaned and validated customer data to ensure accuracy and consistency.

2. SQL Analysis

Calculated churn KPIs, segmentation, and risk patterns using SQL queries.

3. Feature Engineering

Prepared model-ready features using encoding and data transformations.

4. ML Modeling

Built a Logistic Regression model to predict churn probability.

5. Evaluation

Evaluated model using ROC-AUC, precision, recall, and accuracy.

6. Business Insights

Converted predictions into churn risk segments and recommendations.

Technologies Used

Python SQL pandas scikit-learn matplotlib Git & GitHub

Key Insights

  • Customers with short tenure have significantly higher churn rates
  • Month-to-month contracts are more prone to churn
  • High monthly charges increase churn risk
  • SQL helps quickly identify churn-heavy segments before modeling
